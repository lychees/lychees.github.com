On Trust Region Policy Optimization

Posted on 2016-11-19 |

\(\newcommand{\kl}{D_{\mathrm{KL}}}\) Here are the personal notes on some techniques used in Trust Region Policy Optimization (TRPO)

Architecture

A policy is a function from a state to a distribution of actions: \(\pi_\theta(a | s)\). It’s often the case that \(\pi\) is a special distribution parameterized by \(\phi_\theta(s)\). For example, \(\pi_\theta(\cdot | s) \sim \mathcal{N}(\mu_\theta(s), \sigma_\theta^2(s))\), so here \(\phi_\theta(s) = (\mu_\theta(s), \sigma_\theta(s))\). TRPO tries to maximize \(\eta(\theta)\), with the constraint of \(\kl(\pi_\theta(\cdot | s), \pi_{\theta'}(\cdot | s)) \leq \delta\).

Fisher-vector product

An important step in TRPO is to apply Conjugate Gradient algorithm, which requires us to compute the product of a Fisher information matrix and a vector. The Fisher information matrix (FIM) of \(\theta\), is defined by \[ F_\theta(s) = \mathbb{E}\left[ \left(\frac{\partial \log \pi_\theta(x|s)}{\partial \theta}\right) \left(\frac{\partial \log \pi_\theta(x | s)}{\partial \theta} \right)^\top \middle| \theta \right]. \]

The Method Faster to Compute

We now present an effective algorithm to compute \(F v\) given a vector \(v\).

In most cases, \(\pi_\theta(x|s)\) is parameterized by \(\phi_\theta(s)\), i.e., \(\pi_\theta(x|s) = f(x; \phi_\theta(s))\). By some calculus knowledge we have \[ \frac{\partial \log \pi_\theta(x|s)}{\partial \theta} = \frac{\partial \log \pi_\theta(x|s)}{\partial \phi_\theta(s)} \frac{\partial \phi_\theta(s)}{\partial \theta} = \frac{\partial \log f(x; \phi_\theta(s))}{\partial \phi_\theta(s)} J, \] where \(J\) is the Jacobian of \(\phi_\theta(s)\) w.r.t. \(\theta\). We can in turn find the relationship between the FIM of \(F_\theta(s)\) and \(M\), the FIM of \(f\), as \[ F_\theta(s) = J^T \mathbb{E}\left[ \left( \frac{\partial \log f(x; \phi_\theta(s))}{\partial \phi_\theta(s)} \right) \left( \frac{\partial \log f(x; \phi_\theta(s))}{\partial \phi_\theta(s)} \right)^\top \middle | \theta \right] J = J^T M(\phi_\theta(s)) J. \]

To compute \(F_\theta(s) v\), we need to compute \(J^T M(\phi_\theta(s)) J v\). Note that \(Jv\) (or \(J^T v\)) can be computed in a standard forward (or backward) propagation fashion, \(M(\phi_\theta(s))\) can be hand-encoded. The total cost is almost the same as back-propagating twice. However, the code seems a little bit annoying if you don’t have access to code to compute \(J v\) for a vector \(v\). Multiplying \(J^T v\) can be easily embedded into the back-propagation framework: \(J^T v = \nabla_\theta (f \cdot v)\). But I have no idea how to compute \(J v\) in the framework of back-propagation, at least when using TensorFlow.

The Method Easier to Implement

Consider two policy \(\pi_\theta(a | s)\) and \(\pi_{\theta'} (a|s)\). We keep \(\theta'\) fixed, and then consider the Hessian of \(\kl(\pi_{\theta'}(a|s) \|\pi_{\theta}(a|s))\), denoted as \(H\). We use \(H\) to approximate \(F\). Therefore \[ \begin{align} F_{i, j} \approx H_{i, j} &= \frac{\partial^2}{\partial \theta_i \partial \theta_j} \kl(\pi_{\theta'}(a|s) \| \pi_{\theta}(a|s) \\ & =\frac{ \partial^2 }{\partial \theta_i \partial \theta_j} \mathbb{E}_{a \sim \pi_{\theta'}(\cdot | s)}\left[ \log \frac{\pi_{\theta'}(a|s)}{\pi_{\theta}(a|s)} \right]. \\ & = -\frac{ \partial^2 }{\partial \theta_i \partial \theta_j} \mathbb{E}_{a \sim \pi_{\theta'}(\cdot | s)}\left[ \log \pi_{\theta}(a|s) \right] \end{align} \] After find a closed form for the expectation, we can use standard techniques to compute Hessian-vector product by noticing \(Hv = \nabla_\theta (\nabla_\theta f \cdot v)\) which can fully utilize popular auto-differentiation tools. Take the parameterized normal distribution for example. Suppose \(\pi_{\theta}(a|s) \sim \mathcal{N}(\mu_\theta(s), \sigma^2_\theta(s))\), the closed form is \[\begin{align} & -\mathbb{E}_{a \sim \pi_{\theta'}(\cdot | s)} \left[\log \pi_{\theta}(a|s) \right] \\ = & \mathbb{E}_{a \sim \pi_{\theta'}(\cdot | s)} \left[ \frac{(a - \mu_{\theta}(s))^2}{2 \sigma^2_{\theta}(s)} + \log \sigma_\theta(s) \right] \\ = & \mathbb{E}_{a \sim \mathcal{N}(0, 1)} \left[ \frac{(\sigma_{\theta'}(s)a + \mu_{\theta'}(s) - \mu_{\theta}(s))^2}{2 \sigma^2_{\theta}(s)} \right]+ \log \sigma_\theta(s) \\ = & \frac{\sigma^2_{\theta'}(s) + (\mu_{\theta'}(s) - \mu_{\theta}(s))^2}{2 \sigma^2_{\theta}(s)}+ \log \sigma_\theta(s). \end{align}\]

Conjugate Gradient

Here we describe an algorithm to solve a linear equation \(A x = y\) for a symmetric positive definite matrix \(A\), the Conjugate Gradient algorithm (CG).

We define two vectors \(u, v\) are conjugate w.r.t. \(A\), if and only if \(u^T A v = 0\). A set of basis of \(A\) is a set of non-zero vectors \(\{b_1, b_2, \dots, b_n\}\), which are mutual conjugate w.r.t. \(A\).

It’s easy to show that the minimum of a convex function \(L(x) = \frac{1}{2} x^\top A x - x^\top y\) is reached when \(A x = y\). So our goal is to find \(x\) to minimize \(L(x)\) as possible.

\(L(x)\) can be simplified with the help of a set of basis.

\[\begin{align} L(x) & = \frac{1}{2} x^\top A x - x^\top y \\ & = \frac{1}{2} \left(\sum_{i=1}^n \alpha_i b_i \right)^\top A \left( \sum_{i=1}^n \alpha_i b_i \right) - \left( \sum_{i=1}^n \alpha_i b_i \right)^\top y \\ & = \sum_{i=1}^n \left( \frac{1}{2} b_i^\top A b_i \cdot \alpha_i^2 - b_i^\top y \cdot \alpha_i \right). \end{align}\]

Once \(b_i\)s are chosen, only \(\alpha_i\) remains variable. Optimizing each \(\alpha_i\) independently leads to minimizing \(L(x)\).

The \(\alpha_i\) plays a role in a quadratic function, so the minima of \(\alpha_i\) is \[ \alpha_i = \frac{b_i^\top y}{b_i^\top A b_i}. \] Furthermore we need to find a set of basis. This is often done by a process similar to Gram-Schmidt process. For \(0 \leq k < n\), suppose \(b_1, b_2, \dots, b_k\) have already been decided, now we want to find a vector which is conjugate to all of those. We define \[ x_k = \sum_{i = 1}^k \alpha_i b_i. \] If \(A x_k \neq y\), \(r_k = y - A x_k\) should have a component which is conjugate to all \(b_1, b_2, \dots, b_k\). By getting rid of components of \(b_1, b_2, \dots, b_k\), we can find a vector which is conjugate to any known vectors. \(b_{k+1}\) is given by \[ b_{k+1} = r_k - \sum_{i=1}^k \frac{r_k^\top A b_i}{b_i ^\top A b_i} b_i. \] One can easily check \(b_{k+1}^\top A b_i = 0\) for all \(i \in [k]\). (Exercise)

As a summary, the Conjugate Gradient algorithms runs as follows:

  1. initialize \(x_0 \gets 0\);
  2. let \(T\) be number of iterations, can be \(n\) but typically is a constant;
  3. loop over \(k\) from 0 to \(T - 1\): \[\begin{align} r_k & = y - A x_k \\ b_{k+1} & = r_k - \sum_{i=1}^k \frac{r_k^\top A b_i}{b_i ^\top A b_i} b_i \\ \alpha_{k+1} & = \frac{b_{k+1}^\top y}{b_{k+1}^\top A b_{k+1}} \\ x_{k+1} & = x_k + \alpha_{k+1} b_{k+1} \\ \end{align}\]
  4. output \(x = x_T\).

When \(t = n\), CG will produce the exactly solution. However it may be too costly so we often just stop early, i.e., just iterate for a certain number of times.

The algorithm doesn’t seem so efficient since it requires us to restore \(b_1, b_2, \dots, b_k\) in order to compute \(b_{k+1}\). However this issue can be solved by more careful analysis. With the following equations, \[\begin{align} r_k & = r_{k - 1} - \alpha_k A b_k, \\ \beta_k & = \frac{r_k^\top r_k}{r_{k-1}^\top r_{k-1}}, \\ b_{k+1} & = r_k + \beta_k b_k. \\ \end{align}\] \(r_k\) and \(b_k\) can be easily computed, so can \(x_k\).

Note that an advantage of CG is that we don’t need compute the full matrix \(A\). Instead, we only need to know how to compute \(Ax\) given \(x\). For some reasons, it’s unrealistic to compute the whole matrix, e.g., Hession matrix in Hessian-free optimization and Fisher information matrix in Fisher-vector product, where CG becomes a very useful weapon.
